# CodeCoachAI V1 演示说明

## 1. 项目定位

CodeCoachAI 是一个面向 Java 求职者的 AI Java 面试训练与简历项目深挖平台。

项目定位为个人作品集项目，重点展示：

- Java 后端微服务架构能力
- Spring Cloud Alibaba 工程实践
- Vue3 前端页面开发能力
- 题库治理与刷题闭环
- 简历手动录入与项目经历维护
- AI 模拟面试流程控制
- 动态追问、评分点评与面试报告生成
- Prompt 模板管理与 AI 调用日志治理
- Gateway、Nacos、OpenFeign、MySQL、Redis 等基础工程能力

本项目不包装成真实企业系统，不声明真实商业客户或真实生产数据。

---

## 2. V1 功能范围

V1 目标是完成一个可本地启动、可联调、可演示的最小闭环版本。

V1 已覆盖的核心能力：

1. 用户注册、登录、退出。
2. 普通用户与管理员权限边界。
3. 管理端题库管理。
4. 管理端分类管理。
5. 管理端标签管理。
6. 管理端问题组管理。
7. 管理端 Prompt 模板管理。
8. 管理端 AI 调用日志查看。
9. 管理端系统配置基础管理。
10. 用户端题库浏览。
11. 用户端分类、标签、难度筛选。
12. 用户端题目详情查看。
13. 用户端提交答案、掌握状态、收藏、错题简化闭环。
14. 用户端简历手动录入。
15. 用户端项目经历维护。
16. 用户端创建 AI 模拟面试。
17. 用户端 AI 面试房间。
18. 面试开始、回答提交、评分点评、追问或下一题流程。
19. 用户端面试历史。
20. 用户端面试报告。
21. 中文演示数据。
22. V1 浏览器回归测试通过。

以下能力不属于 V1 范围，后续放到 V2 / V3：

- RabbitMQ / RocketMQ 异步任务。

- 学习计划。
- Embedding 语义去重。
- Prompt 版本管理。
- Prompt A/B 测试。
- 完整题型体系。
- 复杂数据看板。

## 8. 关键技术亮点

### 8.1 Spring Cloud Alibaba 微服务骨架

后端采用多模块微服务结构：

- Gateway 统一入口。
- Auth 负责认证。
- User 负责用户信息。
- Question 负责题库、分类、标签、问题组、刷题。
- Resume 负责简历与项目经历。
- Interview 负责面试流程、状态机、报告。
- AI 负责 AI 调用、Prompt、日志。
- System 负责系统配置与后台概览。

---

### 8.2 Gateway + Token + 权限边界

- Gateway 统一转发。
- 普通用户不能访问管理端接口。
- 管理员可以访问后台接口。
- 用户端不误调 `/admin/**`。
- 浏览器回归确认无 `/inner/**` 和旧 `/ai/interview/**`。

---

### 8.3 题库治理基础能力

- 分类管理。
- 标签管理。
- 问题组管理。
- 题目状态管理。
- 用户端只展示启用题。
- 标签筛选使用真实 `tagId`。

---

### 8.4 AI 面试流程控制

- 创建面试会话。
- 生成面试阶段。
- 获取当前问题。
- 提交回答。
- AI 评分点评。
- 控制追问或进入下一题。
- 生成面试报告。

---

### 8.5 Prompt 模板与 AI 日志

- Prompt 不完全写死在代码里。
- 后台可维护 Prompt 模板。
- AI 调用日志可追踪请求、响应和耗时。

---

## 9. 已知边界

V1 可演示，但还不是完整商业化版本。

已知边界：

1. 用户首页统计仍偏基础。
2. 面试大纲生成较简化。
3. AI 评分和追问质量依赖当前模型和 Prompt。
4. 报告分析维度是 V1 简化版。
5. 系统配置对业务流程的驱动有限。
6. Prompt 版本管理未实现。
7. AI 日志筛选和诊断能力较基础。
8. 题型体系未完整实现。
9. 问题组复杂治理未实现。
10. 不支持文件上传、ES、MQ、MinIO、SSE。
