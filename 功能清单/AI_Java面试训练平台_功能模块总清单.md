# AI Java 面试训练与简历优化平台：功能模块总清单

> 项目定位：面向 Java 求职者的个人作品集项目，核心目标是通过 AI 模拟真实 Java 面试官，帮助用户完成八股文训练、简历项目深挖、动态追问、面试报告生成、简历优化和学习规划。
>
> 说明：本文档整理的是“完整功能模块清单”，不代表第一版全部都要开发。后续可以再按 V1 / V2 / V3 拆分开发优先级。

## 6. 题库语义去重模块

这是项目中很有价值的功能点。

普通题库系统只能根据标题完全一致去重，但实际面试题经常存在“问法不同、考察点相同”的情况。

例如：

- 说一下 HashMap
- 介绍一下 HashMap
- 说一下 HashMap 的底层原理
- HashMap 的底层结构是什么

这些问题可能都属于同一个核心问题组。

### 6.1 问题组设计

引入 `question_group` 概念。

一个问题组表示一个“标准问题”或“核心考察意图”。

例如：

```text
问题组：HashMap 核心原理
标准问题：请介绍 HashMap 的底层结构与核心原理
关联问法：
- 说一下 HashMap
- 介绍一下 HashMap
- HashMap 的底层结构是什么
- HashMap 的 put 流程大概是什么
```

### 6.2 题目与问题组关系

一个问题组下面可以挂多个具体题目。

刷题和模拟面试时，可以按问题组去重，避免一场面试里重复问多个本质相同的问题。

### 6.3 去重策略

建议分阶段实现。

#### 第一阶段：手动分组

管理员新增题目时，手动选择所属问题组。

优点：

- 实现简单
- 稳定可靠
- 适合第一版

#### 第二阶段：文本归一化

对题目标题做清洗：

- 去除标点
- 去除空格
- 去除常见语气词
- 弱化“说一下”“介绍一下”“谈谈”“请解释”等问法

用于识别简单重复题。

#### 第三阶段：语义相似度检测

使用 Embedding 对题目生成向量，计算新题和已有题的语义相似度。

相似度高的题目进入“疑似重复”列表。

#### 第四阶段：AI 考察意图判断

调用 AI 判断两个题目是否属于同一考察意图。

AI 输出：

- 是否同义题
- 是否建议合并
- 主要考察点是否一致
- 是否属于追问题
- 是否只是相关题
- 合并建议说明

#### 第五阶段：人工审核合并

系统不自动删除题目，而是提示管理员审核：

- 合并到已有问题组
- 新建问题组
- 标记为追问题
- 标记为相关题
- 忽略

### 6.4 题目关系网络

题目之间不只有重复 / 不重复两种关系。

建议支持以下关系：

- 同义题：SAME_INTENT
- 追问题：FOLLOW_UP
- 相关题：RELATED
- 进阶题：ADVANCED
- 前置题：PREREQUISITE
- 对比题：COMPARE

例如：

```text
主问题：说一下 HashMap
追问题：HashMap 的 put 流程是什么？
进阶题：HashMap 的扩容机制是什么？
相关题：HashMap 为什么线程不安全？
对比题：HashMap 和 ConcurrentHashMap 有什么区别？
```

### 6.5 该模块的项目亮点表达

可以写进简历：

> 设计题库语义去重机制，通过问题组将不同问法但考察意图相同的题目归并到同一标准问题下，避免用户刷题和模拟面试时重复命中同类问题。

如果实现了 AI 判定，可以写：

> 在题目录入流程中引入语义相似度检测和 AI 考察点分析，对新增题目与已有题库进行相似度匹配，识别同义题、相关题和追问题，并提供人工审核合并能力，提升题库质量。

## 11. 简历项目深挖模块

这是项目核心亮点之一。

### 11.1 项目追问点生成

系统从简历项目中分析可追问点。

例如：

如果项目中出现 MySQL：

- 表结构怎么设计？
- 有没有做索引优化？
- 有没有遇到慢 SQL？
- 事务隔离级别怎么选？
- 有没有出现锁冲突？

如果项目中出现 Redis：

- 缓存了什么数据？
- Key 如何设计？
- 过期时间怎么设置？
- 如何保证缓存与数据库一致？
- 如何处理缓存穿透、击穿、雪崩？

如果项目中出现 RabbitMQ：

- 为什么引入消息队列？
- 如何保证消息不丢失？
- 如何处理重复消费？
- 是否使用死信队列？
- 如何保证消费幂等？

如果项目中出现 Spring Cloud：

- 服务如何拆分？
- 服务之间如何调用？
- 配置中心如何使用？
- 网关做了什么？
- 如何限流和熔断？

### 11.2 项目深挖维度

项目深挖可以按以下维度进行：

- 项目背景
- 业务流程
- 用户角色
- 个人职责
- 技术选型
- 数据库设计
- 接口设计
- 缓存设计
- 消息队列设计
- 权限设计
- 异常处理
- 性能优化
- 并发处理
- 线上问题
- 可扩展性
- 项目复盘

### 11.3 项目真实性检查

AI 可以辅助判断简历项目是否写得空泛。

检查内容包括：

- 是否缺少业务背景
- 是否缺少个人职责
- 是否只堆技术名词
- 是否缺少技术难点
- 是否缺少结果数据
- 是否容易被面试官追问穿
- 是否存在过度包装风险

### 11.4 项目深挖结果

系统可以生成：

- 项目可追问点列表
- 项目风险点列表
- 项目表达优化建议
- 可能被问到的问题
- 推荐准备答案

## 14. 动态追问模块

动态追问是模拟真实面试官的关键。

### 14.1 追问触发条件

AI 根据用户回答判断：

- 回答太浅
- 回答错误
- 回答缺少关键点
- 回答中提到了可深挖技术点
- 回答和简历项目相关
- 回答过于模板化
- 回答缺少项目落地

### 14.2 追问策略

支持：

- 基础概念追问
- 底层原理追问
- 项目落地追问
- 场景应用追问
- 对比类追问
- 反问式追问
- 扩展题追问

### 14.3 追问控制

后端需要控制：

- 每个问题最大追问次数
- 每个阶段最大追问次数
- 总面试轮次
- 面试时间限制
- 是否强制进入下一阶段

避免 AI 一直追问导致流程失控。

### 14.4 回答评分

每轮回答可评分：

- 准确性
- 完整性
- 深度
- 项目结合度
- 表达清晰度
- 面试可接受度

可输出 0-100 分。

### 14.5 动态追问示例

用户回答：

> Redis 可以用来做缓存，解决数据库压力大的问题。

AI 可追问：

> 你说 Redis 用来做缓存，那你在项目中具体缓存的是什么数据？缓存 Key 是怎么设计的？如果数据库数据更新了，你怎么保证缓存和数据库一致？

## 33. 核心业务流程

### 33.1 AI 模拟面试流程

```text
用户选择面试类型
  ↓
选择目标岗位、经验年限、行业、难度
  ↓
选择是否基于简历
  ↓
系统生成面试大纲
  ↓
AI 进入第一阶段提问
  ↓
用户回答
  ↓
AI 评分和点评
  ↓
AI 判断是否追问
  ↓
进入下一题或下一阶段
  ↓
面试结束
  ↓
生成面试报告
```

### 33.2 简历项目深挖流程

```text
用户录入或上传简历
  ↓
系统解析简历
  ↓
提取项目经历和技术栈
  ↓
生成项目可追问点
  ↓
用户创建项目深挖面试
  ↓
AI 按项目维度逐步追问
  ↓
生成项目表达问题和改进建议
```

### 33.3 题目录入去重流程

```text
管理员新增题目
  ↓
系统进行文本归一化
  ↓
系统查找相似题
  ↓
AI 判断考察意图
  ↓
展示疑似重复题
  ↓
管理员选择合并、新建、标记追问或忽略
```

### 33.4 面试报告生成流程

```text
面试结束
  ↓
汇总所有问答记录
  ↓
统计各阶段得分
  ↓
AI 生成综合评价
  ↓
系统生成薄弱点列表
  ↓
推荐复习题目
  ↓
生成报告页面
```

## 37. 项目核心亮点总结

这个项目最终可以提炼成以下亮点：

1. **AI 模拟真实 Java 面试官**
   - 支持八股文、项目深挖、行业场景、综合面试。

2. **动态追问机制**
   - 根据用户回答质量决定追问、纠错、换题或提升难度。

3. **简历项目深挖**
   - 从简历项目中提取技术栈和职责，生成有针对性的项目追问。

4. **面试大纲与状态机控制**
   - 后端控制面试阶段和流程，避免 AI 自由发挥导致流程失控。

5. **题库语义去重**
   - 使用问题组归并同一考察意图的问题，避免重复刷题和重复面试。

6. **AI 工程化能力**
   - Prompt 模板管理、AI 调用日志、失败原因记录、限流、重试。

7. **面试报告与学习闭环**
   - 面试后生成结构化报告，识别薄弱点并推荐复习计划。

8. **主流 Java 技术栈覆盖**
   - Spring Boot、MyBatis-Plus、MySQL、Redis、RabbitMQ、MinIO、Elasticsearch、Vue3。

## 38. 最终建议

虽然本文档整理的是完整功能模块，但实际开发时不要一开始全部做。

建议先围绕这条核心主线开发：

```text
用户录入简历
  ↓
创建模拟面试
  ↓
AI 生成面试大纲
  ↓
AI 提问与动态追问
  ↓
用户回答
  ↓
生成面试报告
  ↓
推荐薄弱点和复习方向
```

只要这条闭环完成，这个项目就已经具备作品集价值。其他模块可以按版本逐步增强。
