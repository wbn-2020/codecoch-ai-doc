“AI Java 面试训练与简历优化平台”

核心目标不是做一个普通刷题网站，而是帮助 Java 求职者完成“刷题练习、真实模拟面试、简历项目深挖、薄弱点分析、学习计划生成”这一整套求职准备流程。

下面是我建议你这个项目包含的功能模块。

### 一、用户与权限模块

这是项目的基础模块，主要负责用户登录、用户信息和权限控制。

需要包含用户注册、登录、退出、修改个人信息、上传头像、修改密码等功能。登录认证可以使用 Sa-Token 或 Spring Security + JWT。对于个人项目，我更建议你用 Sa-Token，开发效率高，也容易实现登录校验、角色权限、接口鉴权。

角色可以先设计为普通用户和管理员。普通用户可以刷题、模拟面试、上传简历、生成学习计划。管理员可以管理题库、分类、标签、Prompt 模板、AI 调用日志、系统配置等。

这个模块虽然不是项目亮点，但它能让系统看起来完整，也方便后面做后台管理。

### 二、题库管理模块

题库是整个项目的基础数据来源。你不能让所有问题都由 AI 临时生成，否则问题质量不稳定，也不方便做分类、刷题记录和错题分析。

题库模块需要支持 Java 面试题的增删改查、分类管理、标签管理、难度管理、题型管理、参考答案管理、解析管理、追问问题管理。

题目分类可以包括 Java 基础、集合框架、多线程、JVM、Spring、Spring Boot、MyBatis、MySQL、Redis、RabbitMQ、微服务、分布式、设计模式、项目场景题。

题型可以包括选择题、简答题、场景题、代码题、项目追问题。

每道题最好有这些字段：题目标题、题目内容、参考答案、答案解析、难度、分类、标签、适合经验年限、题型、是否高频、状态。

这个模块对应后台管理系统，管理员维护题库。普通用户可以在前台按分类、难度、标签进行刷题。

### 三、刷题练习模块

这是用户日常使用最多的模块。它的作用是让用户系统性练习 Java 面试题。

需要包含按分类刷题、按难度刷题、随机刷题、高频题刷题、错题重刷、收藏题目、标记掌握程度、查看答案解析等功能。

用户答题后，系统需要记录答题情况，包括是否答对、用户答案、用时、答题时间、题目分类、掌握状态。对于简答题，可以让用户自己标记“已掌握、模糊、不会”，也可以接入 AI 对用户回答进行点评。

这个模块产生的数据很重要，因为后面的 AI 学习计划、薄弱点分析、模拟面试推荐问题，都可以基于用户的刷题记录来生成。

### 四、错题本与收藏模块

这个模块负责帮助用户沉淀自己的薄弱点。

错题本需要记录用户做错的题、不会的题、回答不完整的题。用户可以按分类查看错题，比如 Redis 错题、MySQL 错题、JVM 错题。系统可以支持错题重刷，用户重新答对后可以标记为已掌握。

收藏模块用于保存用户认为重要的题目。比如一些高频场景题、项目追问题、八股文重点题，都可以收藏。

这个模块不复杂，但很实用，也能让后面的学习计划更合理。

### 五、AI 模拟面试模块

这是整个项目最核心的模块。它不要做成简单的“AI 问一句，用户答一句”，而要尽量模拟真实 Java 面试官。

AI 模拟面试可以分为三种模式。

第一种是八股文技术面试。系统围绕 Java 基础、集合、多线程、JVM、Spring、MySQL、Redis、中间件、微服务等方向进行提问。AI 会根据用户回答决定是否追问。如果用户回答很浅，就继续追问原理；如果用户回答不错，就提高难度或切换到场景题。

第二种是简历项目深挖面试。用户选择或上传一份简历，系统解析出项目经历、技术栈、个人职责和项目难点。AI 根据简历内容追问项目细节，比如“你说使用 Redis 优化性能，具体缓存了什么数据？缓存 key 怎么设计？缓存更新策略是什么？如何保证缓存和数据库一致？”

第三种是综合模拟面试。这个模式最接近真实面试。它可以包含自我介绍、八股文、数据库、缓存、中间件、项目深挖、场景设计题、最后总结。适合做成正式面试模式。

模拟面试开始前，用户需要配置面试参数，比如目标岗位、经验年限、行业方向、面试难度、面试时长、面试官风格、是否基于简历。

目标岗位可以是 Java 后端开发、Java 全栈开发、Java 微服务开发。经验年限可以选择一年、三年、五年。行业方向可以选择电商、金融支付、在线教育、SaaS、内容社区、通用业务系统。

面试过程中，系统需要记录每一轮问题、用户回答、AI 点评、问题所属知识点、是否追问、得分、追问原因。

### 六、面试大纲生成模块

这个模块是为了让 AI 面试更像真实面试，而不是随机聊天。

用户开始一场面试后，系统先根据用户配置生成一份面试大纲。比如用户选择“三年 Java 后端，电商行业，综合模拟面试”，系统可以生成这样的面试大纲：开场自我介绍、Java 基础、MySQL、Redis、Spring Boot、项目深挖、电商场景题、总结评价。

每个阶段可以设置题目数量、预计时间、考察重点和难度。后端根据这个大纲控制面试流程，AI 只负责生成具体问题、点评回答和追问。

这个模块很重要，因为它能体现你的系统设计能力。你不是完全把流程交给大模型，而是由后端控制状态和阶段，大模型负责智能生成内容。

### 七、动态追问模块

这是模拟真实面试官的关键能力。

真实面试官不会只问固定题目，而是会根据你的回答继续追问。比如用户回答“Redis 可以用来做缓存”，AI 应该继续追问：“你们项目中缓存的是什么数据？为什么适合缓存？缓存失效策略是什么？如果数据库更新了，缓存怎么处理？”

动态追问需要根据用户回答质量判断下一步动作。回答太浅，就追问细节；回答错误，就指出问题并换角度追问；回答很好，就提高难度；回答和简历项目相关，就追问项目落地细节。

你可以给每轮回答设置一个评分，比如 0 到 100 分。低于 60 分继续基础追问，60 到 80 分问应用场景，高于 80 分问底层原理或项目实践。

这个模块主要依赖 Prompt 设计和 AI 回答评估，但后端需要保存追问关系，比如当前问题是否是追问、父问题是哪一个、追问次数是否超过限制。

### 八、简历管理与解析模块

用户需要能上传或填写自己的简历。简历可以支持两种方式：一种是手动填写，另一种是上传 PDF、Word 文件。

为了降低开发难度，第一版可以先做手动填写简历，后续再支持文件上传解析。手动填写可以包括基本信息、求职方向、技能栈、工作经历、项目经历、教育经历等。

如果支持文件上传，可以用 MinIO 存储文件，后端解析文件文本，再调用 AI 把简历内容结构化。

结构化后的简历信息需要拆成几个部分：个人基本信息、技能栈、工作经历、项目经历、项目技术栈、个人职责、项目亮点、项目难点。

这个模块的核心价值不是上传文件本身，而是为项目深挖面试和简历优化提供数据基础。

### 九、简历项目深挖模块

这个模块是 AI 模拟面试的重点之一。

系统从用户简历中提取项目经历，然后分析每个项目可以被追问的点。比如项目里写了 MySQL，就生成索引、事务、慢 SQL、表设计相关问题；写了 Redis，就生成缓存穿透、缓存击穿、缓存一致性、热点 key 相关问题；写了 RabbitMQ，就生成消息可靠性、重复消费、死信队列、顺序消费相关问题。

项目深挖可以按照几个维度来问：项目背景、业务流程、个人职责、技术选型、数据库设计、缓存设计、接口设计、异常处理、性能优化、线上问题、可扩展性。

这个模块可以帮助用户发现自己简历项目里写得虚、不具体、经不起追问的地方。它非常适合 Java 三年经验岗位，因为三年经验面试通常一定会深挖项目。

### 十、行业场景面试模块

你提到想结合行业信息深挖项目，这个模块就专门做这件事。

系统可以内置几个行业模板，比如电商、金融支付、在线教育、SaaS、内容社区、物流/ERP。每个行业都有自己的常见业务场景和技术关注点。

电商行业可以关注订单、库存、优惠券、支付、购物车、秒杀、超卖、分布式事务。

金融支付可以关注幂等、对账、金额精度、事务一致性、风控、支付回调、数据安全。

在线教育可以关注课程、订单、学习进度、视频播放、题库、考试、防刷。

SaaS 系统可以关注多租户、权限、数据隔离、套餐限制、配置化。

用户选择行业后，AI 在项目深挖时就可以结合行业场景提问。比如用户选择电商行业，AI 可以问：“如果你的项目中涉及库存扣减，如何避免超卖？”“支付回调重复通知怎么保证幂等？”“订单状态流转怎么设计？”

这个功能会让项目明显区别于普通刷题系统。

### 十一、面试报告模块

每场模拟面试结束后，系统需要生成一份结构化面试报告。

报告内容可以包括总分、各模块得分、知识点掌握情况、回答亮点、回答问题、明显短板、项目表达问题、建议复习内容、推荐刷题列表、简历修改建议。

报告页面建议做得漂亮一点，可以用雷达图、柱状图、得分卡片、问题列表、AI 总结。这个页面非常适合作品集展示。

报告不要只给一句“你表现不错”。要具体，比如“Redis 缓存击穿能够说出互斥锁方案，但没有提到热点 key 过期时间随机化和逻辑过期方案”“项目描述中提到接口优化，但没有说明优化前后的数据对比”。

### 十二、AI 简历优化模块

这个模块和项目深挖相关，但目标不同。

项目深挖是模拟面试官追问；简历优化是帮助用户改简历表达。

用户提交简历后，选择目标岗位，比如“Java 后端三年经验”。AI 从以下角度分析简历：技术栈是否匹配、项目描述是否具体、是否突出个人职责、是否有技术难点、是否有量化结果、是否存在过度包装、是否容易被面试官追问。

系统可以输出简历评分、问题列表、优化建议、项目描述改写建议、可能被追问的问题。

这里要注意项目定位：不要做成“AI 帮用户编项目”。你可以强调“基于真实经历优化表达”。这也更适合你自己的情况。

### 十三、AI 题目生成模块

这个模块主要给管理员或用户使用。

管理员可以输入技术点、难度、经验年限、题型，让 AI 生成题目、参考答案、解析和追问问题。比如输入“Redis 缓存一致性，三年经验，场景题”，系统生成一道场景题和参考答案。

AI 生成的题目不要直接进入正式题库，而是进入待审核状态。管理员审核通过后，题目才能发布。这个设计很重要，因为它体现你考虑了 AI 内容不稳定的问题。

普通用户也可以使用“生成专项练习”功能，比如输入“帮我生成 10 道 MySQL 索引优化题”，系统生成临时练习题。

### 十四、学习计划模块

学习计划模块用于把刷题、错题、模拟面试报告串起来。

用户设置目标，比如“30 天准备 Java 三年经验面试”。系统根据用户的错题、薄弱知识点、模拟面试得分，生成学习计划。

学习计划可以拆成每日任务，比如今天复习 JVM 类加载、完成 10 道 MySQL 题、进行一场 Redis 专项模拟面试。用户完成任务后可以打卡。

这个模块让项目形成完整闭环：刷题发现问题，模拟面试暴露问题，学习计划解决问题。

### 十五、Prompt 模板管理模块

这是后台管理里的一个重要模块。

你不要把 Prompt 全部写死在代码里。可以把不同场景的 Prompt 存到数据库里，比如八股文提问 Prompt、项目深挖 Prompt、回答评分 Prompt、简历优化 Prompt、面试报告 Prompt、学习计划 Prompt。

管理员可以在后台修改 Prompt 模板。模板中可以使用变量，比如用户目标岗位、经验年限、行业方向、简历内容、当前问题、用户回答、历史对话。

这个模块能体现你对 AI 工程化的理解，也方便后续调试效果。

### 十六、AI 调用日志模块

这个模块用来记录每次调用大模型的情况。

需要记录调用场景、用户 ID、Prompt 内容、模型名称、请求参数、响应内容、耗时、调用状态、失败原因、token 消耗等。

这样当 AI 返回异常、超时、格式不符合预期时，你可以在后台排查。这个模块也可以配合 Redis 做 AI 调用限流，比如每个用户每天最多调用 30 次。

这是非常加分的工程化模块。

### 十七、系统通知与任务模块

如果你要做得更完整，可以加入任务和通知。

比如简历分析完成后通知用户，学习计划当天任务提醒，模拟面试报告生成完成提醒，AI 生成题目审核完成提醒。

通知可以先做站内通知，不一定要短信或邮件。任务处理可以用 RabbitMQ 异步执行，比如简历解析、面试报告生成、学习计划生成、批量 AI 题目生成。

### 十八、数据看板模块

数据看板可以分为用户端和管理员端。

用户端看板显示刷题数量、连续学习天数、掌握知识点、薄弱知识点、模拟面试平均分、最近学习任务。

管理员端看板显示用户数量、题目数量、AI 调用次数、热门题目、热门知识点、AI 调用失败率、每日活跃用户。

这个模块能让前端展示效果更好，也可以体现数据统计能力。
